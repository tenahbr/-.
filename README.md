Приложение "Графика" (MDK 2.2)
Репозиторий с выполненной практической работой по курсу "Инструментальные средства разработки".

В рамках задания был проведён рефакторинг и оптимизация JavaFX-приложения с применением шаблона проектирования "Фабричный метод" (Factory Method).

📝 Описание проекта
Исходное приложение представляло собой простую форму авторизации с переходом на второе окно. Код контроллера был монолитным и нарушал принципы SOLID, так как смешивал логику проверки данных, навигации и создания графических компонентов.

Что было сделано:

Анализ кода: Выявлены места с жёсткой связанностью (coupling).
Оптимизация: Логика создания окон (Stage, Scene) вынесена в отдельные фабрики.
Паттерн проектирования: Применён шаблон "Фабричный метод" для обеспечения гибкости и расширяемости приложения.
Рефакторинг: Контроллер очищен от инфраструктурной логики, улучшена читаемость кода.
🛠️ Технологии
Язык: Java 17+
Фреймворк: JavaFX
Система сборки: Maven (или стандартный билд IntelliJ IDEA)
Система контроля версий: Git

📁 Структура проекта
src/main/java/org/example/demo1/ — Исходный код на Java.
HelloApplication.java — Точка входа в приложение.
HelloController.java — Контроллер главной формы (оптимизирован).
SceneFactory.java — Интерфейс фабрики (новый).
TaskSceneFactory.java — Реализация фабрики для окна задачи (новая).
src/main/resources/ — Ресурсы приложения.
hello-view.fxml — Главная форма.
task.fxml — Окно задачи.
pom.xml — Файл зависимостей Maven.
