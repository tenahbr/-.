
📱 Приложение «Графика» (MDK 2.2)

Репозиторий с выполненной практической работой по курсу «Инструментальные средства разработки».

В рамках задания был проведён рефакторинг и оптимизация JavaFX-приложения с применением шаблона проектирования «Цепочка обязанностей» (Chain of Responsibility).

📝 Описание проекта

Исходное приложение представляло собой простую форму авторизации с переходом на второе окно. Код контроллера был монолитным: проверка логина/пароля, навигация и логика UI были смешаны в одном классе, что нарушало принципы SOLID.

Что было сделано:

Анализ кода: Выявлены места с жёсткой связанностью (coupling) и нарушением принципа открытости/закрытости (OCP).

Оптимизация: Логика проверки данных вынесена в отдельную, гибкую систему.

Паттерн проектирования: Применён шаблон «Цепочка обязанностей» для обеспечения расширяемости системы валидации.

Рефакторинг: Контроллер очищен от бизнес-логики, улучшена читаемость и поддерживаемость кода.

🛠️ Технологии

Язык: Java 17+

Фреймворк: JavaFX

Система сборки: Maven (или стандартный билд IntelliJ IDEA)

Система контроля версий: Git

📁 Структура проекта

src/main/java/org/example/demo1/ — Исходный код на Java.

HelloApplication.java — Точка входа в приложение.

HelloController.java — Контроллер главной формы (оптимизирован).

Validator.java, EmptyFieldsValidator.java, CredentialsValidator.java — Реализация паттерна «Цепочка обязанностей».

src/main/resources/ — Ресурсы приложения (FXML, CSS).

pom.xml — Файл зависимостей Maven.

✅ Проверка работы

Убедитесь, что у вас установлена Java 17 или выше и Maven.

Откройте проект в IntelliJ IDEA.

Запустите класс org.example.demo1.Launcher или HelloApplication.

Введите в поля Логин и Пароль слово admin.

Нажмите кнопку Далее.

Должно открыться новое окно с заголовком «Задача!».

Если ввести неверные данные, появится окно с ошибкой, текст которой формируется динамически в зависимости от причины (пустые поля или неверный пароль).
